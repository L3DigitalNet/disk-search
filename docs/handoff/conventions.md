# Conventions

Last updated: 2026-07-06

## Quick Reference

| Topic | Convention |
| --- | --- |
| Public repo | No secrets, credential values, private hostnames, private IPs, or internal infrastructure addresses. |
| Source of truth | `docs/specs/hw-radar-master-spec.md`, ADRs, and resolved/open question docs. |
| Stack | Python, Django, HTMX, Scrapy, APScheduler, PostgreSQL + TimescaleDB, uv, Ruff, BasedPyright, pytest, pip-audit. |
| Data model | Keep ADR-0010's identity ladder; do not collapse retail pages into products. |
| Dependencies | Use `uv add` / `uv add --dev` / `uv remove`; commit `uv.lock`. |
| Verification | Run the full gate before claiming code work complete. |
| Local DB | DB tests need TimescaleDB; `HW_RADAR_DB_PORT=5433` is the common workstation override. |
| Handoff | Keep `state.md` short; route durable facts by lifetime. |
| Test fixtures | Never reuse migration-seeded unique keys (see #9). |

## Numbered Conventions

1. **Public hygiene is binding.** Commit only public-safe facts. Credential
   references are allowed; values and private infrastructure details are not.
2. **Spec and ADRs lead implementation.** Significant decisions go through the
   repo's ADR/open-question process and are reconciled back into the master spec.
3. **The canonical entity ladder is fixed by ADR-0010.** Model
   `category -> product_family -> product_model -> product_variant -> listing ->
   offer_snapshot`; aliases are rows, and drive attributes live in satellites.
4. **Pipeline stages stay independently testable.** Build acquisition as
   `fetch -> parse -> normalize -> entity-resolve -> score -> persist -> alert`.
5. **Use the existing Python toolchain.** Ruff owns formatting/linting,
   BasedPyright is strict for `src/`, and branch coverage is enforced by the
   project gate.
6. **DB-backed tests require live TimescaleDB.** Bring the compose DB up first,
   or set the documented port override when the host owns `5432`.
7. **Handoff files are routed by lifetime.** Live state goes in `state.md`,
   current human orientation in `STATUS.md`, open work in `TODO.md`, component
   graph/backlog in `architecture.md`, deployment truth in `deployed.md`, and
   credential references in `credentials.md`.
8. **Do not hand-edit generated docs.** `docs/research/index.md` is generated by
   the research tooling.
9. **Test fixtures must not reuse migration-seeded unique keys.** Migration
   `0005_seed_sources` seeds `SourceSite` rows (`demo`, `wd-recertified`, …);
   a fixture with `normalized_name="demo"` hits the unique constraint. Use a
   per-file key (`rr-demo`, `acc-demo` are existing precedents).
